What is JavaScript?

Marks & Spencer

by Darren Coxall

JavaScript is…

  • A scripting language
  • Object-orientated
  • Distributed by web browsers
  • Scripting languages are languages that can be used to create small, often single file scripts that can perform actions. Usually executed top-to-bottom.
  • Object orientated is a programming paradigm based on objects and their interactions. This is to make modelling behaviour easier as we can design the system around a real life concept. e.g. a car has two wheels can be modelled as a Car object with a value of 2 for it's wheels.
  • Javascript is most commonly distributed via web browsers, many with their own engines that mean they have slightly different support for certain features. This is also what makes it common in websites as we known most visitors have it available simply because they are using their web browser.

JavaScript is NOT…

  • Java
  • Fundamental to HTML
Java is to JavaScript as ham is to hamster

Often for newcomers, javascript and java are confused due to their names. That is where the real similarities end. they are both languages but work very differently. So be sure when searching that you are using JavaScript and not Java.

HTML and it's surrounding languages are all individual components. CSS ties very tightly to HTML but again isn't necessary to view it or have a website. JavaScript is even less necessary. Many people actively disable JavaScript in their browser too. It's not uncommon nowadays for large sites to be un-usable without JavaScript but just a few years ago it was considered poor practice to rely on JavaScript.

How to use it

The most common way to execute JavaScript is to use a script tag in HTML.

The first line demonstrates how to load or include an external file whilst the last 3 lines show how to write javascript directly into the tag. This is similar to CSS and the use of the style tag.

It's also worth noting that this tag can go in the head of the HTML or the body. When in the head it will be executed before any content is actually available so a common technique is to put script tags at the very bottom of the body tag to ensure there is content on the page before executing.

To really use and understanding JavaScript you need to develop some fundamental programming skills. These concepts are often cross functional and apply to all languages and are the building blocks of solving problems with code.

There is a lot here so I'm going to give a basic overview of each. When combined they become LEGO blocks to code that can be used to solve a huge variety of problems, not just in JavaScript but any other programming language.

Variables

Think of a variable as a bucket that can hold some information. This information can be changed later making it variable. 


var myFavouriteNumber = 26;

All the examples shown will be written in JavaScript. This example is me creating a variable named myFavouriteNumber and setting it to the number 26. The semi-colon is important as it tells JavaScript that it's the end of the line.

Data Types

There are many different ways to represent information. 


var myFavouriteNumber = 26;  // Integer
var myFavouriteFloat  = 1.3; // Float

var myName = "Darren Coxall"; // String
var myLotteryNumbers = [10, 12, 18, 34, 52, 56, 50]; // Array
var groceryList = [
  "cheese",
  "eggs",
  "ham"
];

Data types have specific characteristics that make them better for modelling certain types of information.

  • An integer has no concept of decimal places or fractions
  • A float is an approximation of a decimal number so rounding can cause issues
  • A String is great for text but is less useful for numeric values
  • An Array is great for storing a list or collection of information but what happens if it's empty?

They all have pros and cons. Also please note how an interpretted line can span multiple visible lines only ending after the semi colon.

Functions

A function allows you to create re-usable code. 


function onePlusOne() {
  return 1 + 1;
}

onePlusOne() + onePlusOne(); // => (2 + 2) 4

A function, once defined can be called multiple times which will make it re-execute. In this example I define a function called onePlusOne and then I add the result of calling it twice, together.

Just to confuse matters, also note how a semi-colon isn't necessary whyen defining a function in this way.

Functions cont…


// a variable function
var onePlusOne = function() {
  return 1 + 1;
};

onePlusOne() + onePlusOne(); // => (2 + 2) 4

Now here you see we can actually store the function itself within a variable. When doing it this way you should be using the semi colon at the end of the function.

Parameters & Arguments

Functions without parameters don't provide much flexibility. But when we can add different inputs we get a lot more. 


function multiply(x, y) {
  return x * y;
}

multiply(2, 3);   // => 6
multiply(10, 10); // => 100

multiply(2, multiply(1, 10)); // => 20

Within the brackets you can define the names of your parameters. In this case, x and y. These will then operate like variables within the function allowing you use them as if they were the item stored within.

Objects

Objects are simply an instance of a data type or class (a custom data type). In JavaScript you can build your own objects using curl braces. 


var aboutMe = {
  name:   "Darren Coxall",
  age:    26,
  height: [6, 0],
  sayHello: function(name) {
    return "Hello, " + name;
  }
};

aboutMe.name;   // => "Darren Coxall"
aboutMe.height; // => [6, 0]

aboutMe.sayHello("Jane"); // => "Hello. Jane"

To access data within an object, or to call a method on an object we use what is known as dot notation. This is why there is a period or dot after the object name but before the attribute/method name.

Methods

Commonly confused with functions. The difference is that a method has access to it's own object. 


var aboutMe = {
  name:     "Darren Coxall",
  sayHello: function(name) {
    return this.name + " says hello to " + name;
  }
};

aboutMe.sayHello("Jane"); // => "Darren Coxall says hello to Jane"

In this example the keyword "this" allows us to use other information from the same object. This is an important distinction between functions and methods. Functions are stateless, they operate on inputs and outputs whereas Methods have state provided by the object itself.

Other terms

  • DOM
  • jQuery
  • Node
  • DOM stands for Document Object Model. When the browser renders HTML it is building a tree of tags into it's memory. This is the DOM. Each tag is an object that models the the document when combined.
  • jQuery is a popular library written in javascript for manipulating websites and the DOM. It helps to hide the complexity in supporting the different browsers whilst providing lots of utility functions for working with particular data types.
  • Node is some software that let's you run JavaScript on a server. It also gives the language access to things it otherwise doesn't get to interact with such as files on your computer.
